# KB-013: User Roles & Permissions in FlowCRM

**Product:** FlowCRM  
**Category:** Administration  
**Last Updated:** December 2025  
**Article ID:** KB-013

## Summary
This article explains the built-in roles, permission model, and how to create custom roles to control access in FlowCRM.

## Built-in Roles
- **Admin:** Full access to settings, billing, integrations, and user management.
- **Manager:** Can view and edit team records, run reports, and manage pipelines.
- **User:** Day-to-day access to contacts, opportunities, and activities.
- **Read-only:** View-only access for auditors or stakeholders.

## Creating Custom Roles
1. Go to **Settings â†’ Users & Roles**
2. Click **"Create Role"**
3. Choose granular permissions: Contacts, Deals, Reports, Integrations, Admin.
4. Assign role to one or multiple users.

## Permission Best Practices
- Use least privilege: grant only required permissions.
- Create a temporary role for contractors with limited access.
- Audit roles quarterly.

## Troubleshooting Access Issues
- Ensure the user is assigned to the correct role and team.
- Check feature-flag gated features (e.g., beta features may require special access).
